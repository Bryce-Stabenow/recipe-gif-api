GIF Recipe API

This API was developed by Bryce Stabenow to provide a useful interface for those looking to get GIFs of creative recipes online.

Result Format

****Must use iframe to enable playback****

The result of basic requests is a JSON file containing whatever has been requested. A seach, for example, for falafel would return the following:

{
'name': 'Falafel',
'gifLink': 'https://gfycat.com/ifr/EnergeticNecessaryBellfrog',
'cuisine': 'Israeli',
'meal': 'Lunch/Dinner',
'mealCourse': 'Main Course'
}

The resulting data provides a gif link that is accessible using an iframe. Due to copyright laws, providing the GIFs themsleves proves messy, so this is the best solution for both you and me.

To get a random recipe from the API, simply send a GET request to https://gif-recipe-api.herokuapp.com/api/random

Get All Recipes

To get a list of all recipes from the API, simply send a GET request to https://gif-recipe-api.herokuapp.com/api/allRecipes. The API currently has ~106 different recipes to choose from, so the resulting JSON is quite long.

Searching Recipes

To seach for a particular recipe, you can send a GET request to https://gif-recipe-api.herokuapp.com/api/search/<<recipename>>. If the request cannot find a matching response in the current database, it will return the following blank JSON:

{
'name': 'Recipe Not Found',
'gifLink': '',
'cuisine': '',
'meal': '',
'mealCourse': ''
}

There currently is no "closest answer" response, but this is something that I hope to implement in future versions of this API. If you would like to help contribute to this project, for the Github repo and send me a pull request!

Filtering Recipes

Filtering by be via the three object props listed for each recipe.

Current endpoints:

  • Filter by Cuisine: http://localhost:3000/api/filter/cuisine/<<Nationality>>
  • Filter by Meal: http://localhost:3000/api/filter/meal/<<Breakfast, Lunch, or Dinner>>
  • Filter by Course: http://localhost:3000/api/filter/course/<<Main Course, Side Dish, Appetizer, Dessert, or Drink>>

The currently supported nationalities for the 'cuisine' filter are American, Chinese, Japanese, Korean, Turkish, Italian, Ethiopean, Isreali, Filipino, Swedish, Indian, Mexican, Irish, Betelgeusian, Mongolian, Arabian, German, Vietnamese, African, French, Greek, Russian, and Unknown.
